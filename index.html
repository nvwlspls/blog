The easiest things would be to do nothing. However I have decided that I want a blog and that requires doing something.

There are several ways to do this. I could sign up for something on wordpress, ghost or any other number of services for free.

However I am taking a different approach, at least for now. I want as much of this site to be from my code, or at least code that I understand, as possible.

The easiest way to achieve this is to keep it in raw html and code everything out. Eventually this will get tedious.

I am going to use this enumerate my half ass plan for this blog.

  • Get this basic page styled
    • My aestetic inspration:
    • Brutalist Web Design

  • Publish this basic site on github pages
  • Publish this basic site on other static site hosts

  • Create something that will convert Markdown files to html so I can write posts in markdown and not html

The 'user' should see minimal incremental updates with little to no idea of what changes are taking place under the hood.
